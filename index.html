  • GitHub
  • Simple

    Python-like syntax with fewer keywords than C.

  • Explicit

    No hidden memory allocations, not even on the stack. No stack overflow.

  • Thorough

    Full type inference incorporating formal verification. No generics or templates.

  • Interpretable

    Compile-time evaluation of syscalls. No build scripts. No macros.

Planned keywords

  1. if: Python's if without the :.
  2. loop: Rust's loop without the brackets.
  3. def: Python's def without the :.
  4. in: The arguments given to a function.
  5. out: The return of a function.
  6. comptime: Forces evaluation at compile-time.
  7. require: A condition to prove with formal verification at compile-time.
  8. assume: A condition assumed to be true at compile-time.
  9. assert: A condition that calls exit if false at run-time.

Examples

One

def add
lhs = in[0] # 1
rhs = in[1] # 2
out = rhs + lhs # 3
x := add 1 2 # 3
comptime write 1 x
require x = 3 # true
assume x = 4
require x = 4 # true
assert x = 4 # false, but this statement will get removed by the compiler due to the assume.
