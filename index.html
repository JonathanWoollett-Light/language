Keywords

  1. if: Python's if without the :.
  2. loop: Rust's loop without the brackets.
  3. break: Jumps out of the current loop.
  4. def: Python's def without the :.
  5. in: The arguments given to a function.
  6. out: The return of a function.
  7. require: A condition to prove with formal verification at compile-time.
  8. assume: A condition assumed to be true at compile-time.

Tentative keywords

  1. comptime: Forces evaluation at compile-time.
  2. assert: A condition that calls exit if false at run-time.
  3. verify: Like require but does not exhaustively check the condition. verify x < 2 will apply a number of steps of gradient descent to search for the minium of x to verify the condition. While this is not formal verification it provides an effective and almost always correct mechanism to test conditions at compile time that would be too computationally expensive to exhaustively prove.
  4. abstain: Resets the understood domain of a variable at compile to any value.
